    Dictionary

    • 0fuse Console - A console with no CPU fuses burned, allowing it to boot ANY nand image for the board.

    • XeLL image - a NAND image without the CE, CF and CG; Directly from CD to XeLL in boot.

    • Shadowboot image - A NAND image designed for devkits, to easily reboot into an entirely new NAND image from storage. Has been reversed to allow shadowbooting on glitched retail consoles.

    • Retail image - A legitimate retail NAND image

    • Devkit image - A legitimate devkit NAND image

    • 1f - The original 4xxx kernel exploit rebooter images; Reboots into XeLL, Requires JTAG

    • 2f - The modern xeBuild rebooter images; Reboots into FreeBoot, Requires JTAG

    • Glitch1 - A Single-CB hacked image, Requires RGH1 or JTAG

    • Glitch2 - A Split-CB hacked image, Requires RGH2 or 1.2

    • Glitch3 - A Split-CB hacked image with a intermediate CB (CB_X). We (GGX) handle this by combining it with CB_B. Requires RGH 3 or 1.3

    • Glitch*m - A glitch image with emulated eFuses

    • Glitch*s - A glitch image with no patchslots

    • DevGl - A Devkit image that loads a retail kernel with Glitch patches

    • RGL - A 50/50 image, patches retail CD to load devkit kernel

    • JTAG - The Kernel SMC hack used by 1f and 2f images

    • RGH 1 - The original reset glitch, for single-cb.

    • RGH 2 - The newer, but worse, reset glitch for split-cb. Designed for Slims before 1.2v2 was found

    • RGH 3 - The newest, chipless glitch.

    • RGH 1.2 - Porting RGH 1 to Split-CB, v2 ports it to slims

    • RGH 1.3 - RGH 1.2v2, but using an intermediate CB for a more reliable glitch

    • xeBuild patch - 3 or 4 section Builder bootloader patch file

    • Loaderpatch - 1 section Builder patch file

    • RGLoader patch - Builder or Memory 1 section library patch file

    • CDXell patch - Memory multi section patch filesystem patch
