    NAND Patches

    My personal research on xeBuild Patches (BIN), RGLoader Patches (RGLP) and LoaderPatch Patches (TXT)

    xeBuild Patches

    The binaries are made built from assembly patch files. A full repository of xeBuild patch sources with a build script can be found on Mitchell Waite's GitHub.

    Binaries

    xeBuild patchsets (XEPATCH0 header?) are made up of 3 or 4 different patch sections, seperated by FFFFFF.

    Modern JTAG - 1BL - CB - CD - Kernel / Hypervisor

    Reset Glitch (GG) - CB / CB_B - CD - Kernel / Hypervisor

    Individual patches have no header data and are just a compiled asm bin.

    Patch Sections

    Invidual sections are treated differently; CB/CB_B/CD are read and applied directly by xeBuild, whereas Kernel / Hypervisor patches are applied by the patched CD on boot.

    xeBuild patch sections are structured as followed:

    Offset Type Description
    0x0 Unsigned 32-bit integer Target Address
    0x4 Unsigned 32-bit integer Target Amount
    0x8 Unsigned 32-bit integer array Patch Data
    From: Emma's Research

    CD / KHV Patches

    Also Read: CD Patches and Kernel Patches

    Also See: Raw xeBuild CD/4BL and KVH patches

    CD patches work differently, not only do they disable panics and emulate fuses, they contain a patching engine that loads and applies Kernel/Hypervisor patches in memory. This way, the base 1888 kernel and hypervisor can remain untouched, and all modifications happen in memory.

    General use-cases

    • CB / CB_B -> Enable post codes / uart, virtual fuses, break trust chain for reset glitch
    • CD -> Load and apply kernel / hypervisor patches
    • Kernel / Hypervisor -> Unsigned code execution, debugging, the actual OS modifications

    DashLaunch Patches

    DashLaunch is the companion in-hypervisor runtime patcher for xeBuild. Requires CD to have already patched Kernel / Hypervisor for unsigned code execution. Reads launch.ini from the flashfs root, and applies listed plugins / patches.

    Loaderpatch Patches

    Loaderpatch (By stoker25) is the bootloader patcher for RGBuild. As simple as it gets, just asm stored in a text file, uncompiled.

    I plan to add support for Loaderpatch to gxBuild, most likely through an external app that converts them to xebuild binaries.

    Loaderpatch Sources: RGLoader-Patches

    RGLoader Patches

    RGLP Sources: RGLoader-Patches

    RGBuild has a custom patching format for their Kernel / XAM / Hypervisor patches: RGLP. I believe the latest version of xeBuild supports these patches.

    This format can be applied directly to a loader or applied in-memory by RGLoader.xex.
